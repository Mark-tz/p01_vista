### 2.2 核心交互流程

#### 地图可视化数据流

```
                      ┌──────────────┐
                      │  地图服务     │
                      │  (Map Svc)   │
                      └──────┬───────┘
                             │ REST: 地图数据 + 点位 + 围栏
                             ▼
┌──────────────────────────────────────────────────────┐
│                    前端地图引擎                        │
│  ┌─────────┐   ┌──────────┐   ┌────────────────┐   │
│  │ 底图渲染 │   │ 要素叠加  │   │ 实时数据绑定    │   │
│  │(瓦片/模型)│   │(点位/围栏)│   │(WebSocket)     │   │
│  └─────────┘   └──────────┘   └────────────────┘   │
└──────────────────────────────────────────────────────┘
                             ▲
                             │ WebSocket: 位置坐标 (10Hz)
                      ┌──────┴───────┐
                      │  监控服务     │
                      │ (Monitor Svc)│
                      └──────────────┘
```

#### 机器人状态更新频率

| 数据类型 | 更新频率 | QoS | 传输方式 |
|----------|----------|-----|----------|
| GPS/位姿 | 10 Hz | 0 | MQTT → WebSocket |
| 电量 | 0.1 Hz | 1 | MQTT → WebSocket |
| 网络质量 | 0.2 Hz | 0 | MQTT → WebSocket |
| 运行模式 | 事件触发 | 1 | MQTT → WebSocket |
| 健康状态 | 0.05 Hz | 1 | MQTT → WebSocket |

### 2.3 关键设计要点

- **地图渲染性能**：大规模点位使用 Canvas 绑定替代 DOM 标记，单图层 10000+ 点位无压力
- **轨迹回放**：后端按时间范围查询 InfluxDB，前端使用 requestAnimationFrame 插值平滑播放
- **视频流管理**：同时最多开 4 路 WebRTC 流，超出时自动降级为低码率或暂停非焦点窗口
- **WebSocket 房间**：按场站(site) + 楼层(floor) 分房间，前端只订阅当前视图相关数据

### 3.4 充电调度策略

```
电量检查 (每个巡检点完成后)
    │
    ├── 电量 > 30%  → 继续执行
    │
    ├── 20% < 电量 ≤ 30%
    │   ├── 剩余点位预估可完成 → 继续
    │   └── 预估不可完成 → 暂停任务，返回充电
    │
    └── 电量 ≤ 20%  → 立即返回充电
        │
        充电至 90% → 恢复任务从断点继续
```

### 5.2 告警处理流水线

```
数据源                  Kafka Topic              告警服务
                                              
Device Svc ──►  ┐                         ┌── 规则匹配
Monitor Svc ──► ├► vista.alarm.raw ──────►├── 去重/合并
AI Svc ──────►  ┤                         ├── 级别判定
Task Svc ────►  ┘                         └── 通知分发
                                               │
                                    ┌──────────┼──────────┐
                                    ▼          ▼          ▼
                              WebSocket     短信/邮件    Webhook
                              (实时弹窗)    (通知)      (第三方)
```

### 5.3 告警级别定义

| 级别 | 名称 | 响应时限 | 通知方式 | 示例 |
|------|------|----------|----------|------|
| L1 | 紧急 | 即时 | 弹窗 + 短信 + 电话 | 机器人失联、气体泄漏 |
| L2 | 重要 | 15 min | 弹窗 + 短信 | 温度超限、AI 识别异常 |
| L3 | 一般 | 1 hour | 弹窗 + 邮件 | 电量低、网络信号弱 |
| L4 | 提示 | 8 hours | 仅平台 | 任务超时、传感器偏差 |

### 5.4 告警合并策略

- **时间窗口合并**：同源同类告警在 5 分钟窗口内合并为一条
- **抖动抑制**：阈值告警在恢复/触发之间需保持 30s 稳定态才确认状态变更
- **升级机制**：L3 告警持续 30min 未确认自动升级为 L2

### 6.3 安全约束

| 约束 | 说明 |
|------|------|
| 速度限制 | 远程操控模式最大速度为自主模式的 50% |
| 心跳检测 | 3 秒无指令自动停止运动 |
| 权限分级 | 移动控制需运维角色，机械臂需管理员角色 |
| 单人操控 | 同一时间只能有一人操控一台机器人，抢占需确认 |
| 紧急停止 | E-Stop 指令走独立高优先级通道，端到端 < 100ms |
