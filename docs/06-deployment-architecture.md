## 1. 部署模式总览

系统支持三种部署模式，适应不同客户场景：

| 模式 | 网络环境 | 适用场景 | 特点 |
|------|----------|----------|------|
| **局域网部署** | 内网隔离 | 电厂/化工厂等高安全要求 | 全部组件本地化，零外网依赖 |
| **公网云部署** | 公有云 | 多站点集中管理 | 弹性扩展，统一管理多个场站 |
| **混合部署** | 边缘+云 | 大型集团多站点 | 边缘低延迟操控 + 云端统一管控 |

**边缘节点部署的服务**（保证断网可用）：
- 任务调度服务（本地任务继续执行）
- 实时监控服务（本地视频/状态不中断）
- 远程操控服务（本地操控零延迟）
- AI 推理服务（边缘推理，无需上云）
- EMQX（设备通信不依赖外网）

**云端负责**：
- 统一用户管理
- 跨站点数据汇聚
- 集中告警管理
- 历史数据分析与报表
- OTA 统一分发

## 5. 容器编排

### 5.1 K3s 选型理由

| 特性 | K3s | K8s |
|------|-----|-----|
| 二进制大小 | ~100MB | ~1GB+ |
| 最低资源 | 1C512M | 2C2G |
| 安装复杂度 | 一键脚本 | 较复杂 |
| 边缘适配 | 原生支持 | 需额外适配 |
| 功能完整性 | 满足需求 | 完整 |
| 生产可用 | CNCF 认证 | 标准 |

### 5.2 命名空间规划

```
vista-system          # 系统组件（Gateway, IAM）
vista-business        # 业务服务（Task, Device, Map, Alarm...）
vista-ai              # AI 服务（GPU 节点亲和性）
vista-media           # 媒体服务（高带宽需求）
vista-middleware       # 中间件（PostgreSQL, Redis, InfluxDB, Kafka, EMQX）
vista-monitoring       # 可观测性（Prometheus, Grafana, Loki, Jaeger）
```

## 9. 备份与容灾

### 9.1 备份策略

| 数据 | 方式 | 频率 | 保留 | 恢复目标 |
|------|------|------|------|----------|
| PostgreSQL | pg_dump + WAL 归档 | 全量: 每日; 增量: 实时 | 30 天 | RPO < 1min, RTO < 30min |
| Redis | RDB + AOF | RDB: 每小时; AOF: 实时 | 7 天 | RPO < 1s, RTO < 5min |
| InfluxDB | 内置备份 | 每日 | 7 天 | RPO < 1day, RTO < 1h |
| MinIO | 跨节点复制 | 实时 | — | RPO ≈ 0, RTO < 10min |
| ETCD (K3s) | 快照 | 每 6 小时 | 7 天 | RTO < 30min |

### 9.2 容灾等级

| 模式 | 容灾方案 | 切换时间 | 数据丢失 |
|------|----------|----------|----------|
| 局域网单站点 | 主备切换 (Keepalived) | < 30s | < 1min |
| 云部署 | 多可用区 + 数据库主从 | < 60s | ≈ 0 |
| 混合部署 | 边缘自治 + 云端热备 | 边缘: 0 (自治); 云: < 60s | 边缘数据延迟同步 |
